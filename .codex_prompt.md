# Codex咨询任务：评测+可观测+Hook方案设计

你是一位资深的AI系统架构师，经验丰富，做事沉稳严谨。请为以下航空气象AI Agent项目设计一套完整的【自动化评测流水线 + 全链路可观测性 + Git Hook质量门禁】方案。

## 项目背景

航空气象智能分析系统，核心架构:
- **后端**: FastAPI + LangGraph ReAct Agent
- **LLM**: 百度千帆/DeepSeek/OpenAI 多Provider自动切换
- **工具层**: 8个气象工具 (fetch_metar, parse_metar, get_flight_rules, assess_risk, get_approach_minima, format_visibility, fetch_taf, get_full_weather)
- **API**: V3 Agent模式, POST /api/v3/chat, 带session记忆
- **前端**: Next.js 15
- **已有评测**: D1-D5五维指标 (D1规则映射≥95%, D2角色匹配≥85%, D3安全边界=100%, D4幻觉率≤5%, D5越权率=0%)
- **环境**: macOS M3 + Docker, Python 3.11, LangGraph 0.2.28

## 你需要详细设计的三个方向

### 一、自动化评测流水线

1. 评测框架选型: DeepEval (比RAGAS更适配非RAG Agent场景)
2. 扩展评测指标: D6工具调用准确率≥90%, D7上下文精度≥85%, D8端到端延迟P95<8s
3. Golden Set数据集设计 (51条已有 + 扩展边界case)
4. CI/CD Gate: pre-push hook + GitHub Actions, 指标不达标拒绝push
5. 评测报告格式

### 二、全链路可观测性

1. OpenTelemetry标准采集
2. 追踪字段: prompt内容, 补全输出, token消耗(I/O分开), 端到端延迟, 请求失败率, tool_call链路
3. Langfuse (24.9k⭐) 做LLM追踪, 自部署Docker
4. Phoenix (9.3k⭐) 做幻觉率趋势
5. Prometheus + Grafana 做指标存储+可视化
6. OTel Collector 统一分发
7. 告警规则: 4类告警
8. Docker Compose一键部署

### 三、三层Hook质量门禁

Layer 1 安全网: PreToolUse拦截危险命令, 锁敏感文件, PostToolUse自动格式化
Layer 2 改后即测: 自动跑关联测试, 独立reviewer subagent审查, PR强制全量测试
Layer 3 规范日志: pre-commit lint+typecheck, 操作日志.jsonl, auto-commit LLM生成提交信息
